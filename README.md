# Watermark Experiment Code

Note: `watermark_reliability_release/` is copied from
`https://github.com/jwkirchenbauer/lm-watermarking/tree/main` and kept here as a
local dependency for the experiment code.

This repository contains the code needed to generate watermark experiment data,
quality metrics, attack data, and figure data/plots.

## Contents

- `experiment_config.py` - model and dataset configuration.
- `experiment_scheduler.py` - schedules the default 3x3 experiment grid.
- `run_experiment_combined.py` - runs one watermark generation experiment.
- `solver_parameter_generate.py` - regenerates `optimal_power_results_combined.csv`.
- `generate_quality_metrics.py` - computes BLEU, ROUGE, and BERTScore CSV data.
- `run_attack.py` - runs attack experiments and aggregation.
- `theorytools.py` - shared theory and parameter utilities.
- `watermark_reliability_release/` - local watermark processor dependency.
- `figure_generation/notebooks/` - figure-generation notebooks with outputs cleared.

## Default Scope

The default scheduler runs 3 models x 3 datasets:

- Models: `opt-125m`, `gpt2`, `pythia-160m`
- Datasets: `c4`, `lfqa`, `wikipedia`

`gemma-2-9b` is available in `experiment_config.py` for single runs, but it is not
included in the default scheduler.

## Common Commands

Regenerate the KL-front parameter CSV:

```bash
python solver_parameter_generate.py
```

Run the default experiment scheduler:

```bash
python experiment_scheduler.py --output-dir results_final
```

Run selected methods only:

```bash
python experiment_scheduler.py --methods GRID,KLFRONT --output-dir results_final
```

Generate quality metric CSV files after experiment results exist:

```bash
python generate_quality_metrics.py --results-dir results_final --output-dir quality_metrics
```

Show scheduler options:

```bash
python experiment_scheduler.py --help
```

Run attack aggregation or attack generation:

```bash
python run_attack.py --help
```

## Generated Data

`optimal_power_results_combined.csv` is generated by
`solver_parameter_generate.py` and keeps these columns:

```text
alpha,n_samples,gamma,delta,base_gamma
```

Experiment JSON files are written under `results_final/` by default. Quality
metric CSV files are written under `quality_metrics/` by default.
